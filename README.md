<h1>Módulos do Projeto</h1>
<h2>1. Front-End (Interface Fluida & SPA)</h2>
• Desenvolvido com uma estética minimalista e geométrica (tons escuros de alta tecnologia combinados com verde esmeralda para eficiência energética).
• Navegação SPA Nativa: Transições e animações suaves (cubic-bezier) ao alternar entre as abas (Início, Simulador e Metas ODS 7) sem recarregar o navegador.
• Mapeamento Visual: Integração com a biblioteca Chart.js para renderizar um gráfico de rosca dinâmico que aponta os maiores gargalos de consumo em tempo real.
• Micro-interações: Sistema de notificações internas (toasts) para feedbacks instantâneos de inserção e remoção de ativos.

<h2>2. Modelagem Java (Orientação a Objetos)</h2>
• Implementação rigorosa dos pilares de POO (Encapsulamento, Composição e Abstração) mapeando o cenário real de uma residência:
• Uma instância de Usuario possui uma composição de múltiplos objetos da classe Comodo.
• Cada Comodo gerencia sua própria lista de objetos da classe Aparelho.
• Processamento centralizado das regras de negócio e cálculo escalável do impacto financeiro baseado em tarifas locais configuráveis.

<h2>3. Motor em C (Estrutura de Dados)</h2>
• Código de baixo nível projetado para simular o processamento core e a otimização de recursos de hardware:
• Lista Encadeada Dinâmica: Uso de ponteiros (*prox) para interligar os aparelhos cadastrados na memória.
• Gerenciamento de Memória Ativo: Alocação em tempo de execução via malloc() ao inserir um novo ativo e liberação cirúrgica via free() ao excluir ou encerrar o sistema, eliminando riscos de vazamento de memória (memory leaks).
